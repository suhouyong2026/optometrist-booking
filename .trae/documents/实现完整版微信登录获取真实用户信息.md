## 完整版微信登录流程

### 当前问题
目前使用的是简化版微信登录，只是生成模拟的openid，没有调用微信API获取真实的用户头像和昵称。

### 实现完整版需要以下步骤：

#### 1. 微信公众账号配置（已完成的部分）
- 已配置网页授权域名 ✅
- 已获取 AppID ✅

#### 2. 需要新增/修改的内容

**前端修改 (index.html 和 wechat-callback.html):**
- 微信授权时请求 `snsapi_userinfo` 权限（已配置）
- 在回调页面用 code 换取 access_token
- 用 access_token 获取用户基本信息（头像、昵称）

**云函数修改 (booking-system/index.js):**
- 新增 `/api/wechat/access_token` 接口：用 code 换取 access_token
- 新增 `/api/wechat/userinfo` 接口：用 access_token 获取用户信息
- 修改回调处理逻辑，保存真实用户信息

**需要准备的材料：**
- 微信 AppSecret（需要在公众号后台获取，并配置到云函数环境变量）

#### 3. 具体实现步骤

1. **获取 AppSecret**
   - 登录微信公众号后台
   - 开发 → 基本配置 → 获取 AppSecret
   - 将 AppSecret 配置到腾讯云函数环境变量

2. **修改云函数**
   - 添加获取 access_token 的接口
   - 添加获取用户信息的接口
   - 修改回调处理，调用真实微信API

3. **修改前端回调页面**
   - 在 wechat-callback.html 中调用云函数获取真实用户信息
   - 保存到 localStorage

#### 4. 安全注意事项
- AppSecret 不能暴露在前端代码中，必须通过云函数调用
- 用户授权后，微信会返回临时凭证 code，用 code 换取 access_token

### 需要修改的文件
1. `functions/booking-system/index.js` - 新增微信API调用接口
2. `wechat-callback.html` - 修改回调处理逻辑
